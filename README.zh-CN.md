## Kinal-ThirdParty

## [English version](README.md)

这个仓库用于存放 Kinal 依赖的第三方源码、归档包或较大的预编译资产。

它和主仓库 `Kinal` 分开，是为了让语言本体、运行时、工具链源码保持更小、更容易克隆，也更适合代码搜索和维护。

[此处可以通往 Kinal 主仓库](https://github.com/Kinal-Lang/Kinal)

### 当前结构

- `llvm/`
  - `src/`：LLVM 上游源码快照或解压后的源码树
  - `manifests/`：用于自举和下载的元数据清单
- `civetweb/`
  - `include/`：公共头文件
  - `src/`：固定版本源码
  - `manifests/`：原生构建规则和版本说明

### 设计意图

- 让 Kinal 源码和第三方资产彻底分仓。
- 让 `Kinal` 可以通过同级目录自动发现本仓库：
  - `../Kinal-ThirdParty`
- 避免把大体积归档包和预编译工具链塞进主语言仓库历史。

### 关于生成产物

这个仓库更适合保存：

- 上游源码
- 头文件
- 版本清单和下载元数据

不适合长期把机器本地的预编译产物直接作为仓库结构的一部分保存下来。

如果构建脚本需要下载工具链或生成原生中间产物，更合适的放置位置应当是被忽略的目录，例如：

- `.cache/`
- `artifacts/`
- 或者构建脚本自己维护的工作目录

### 说明

- 这个仓库只放第三方源码和二进制资产。
- 更推荐放源码快照和清单，而不是长期跟踪预编译输出。
- Kinal 自己的应用、运行时逻辑和工具应该放在 `Kinal` 仓库里。
- 即使某个依赖以后继续膨胀，只要它仍属于外部资产，放在这里也比混进主仓库历史更健康。
